## Adjustment Rules

### 1. Volume Ceiling Per Sport

When substituting one sport for another, the substitute volume is capped:

```
max_additional = current_weekly_volume_in_sport × 0.15
```

Example: if the athlete currently does 180min swim/week, max additional swim = 27min. You cannot dump 240min of missing bike onto swim.

Accept the volume deficit. A week with lower total volume is better than overloading a substitute sport.

### 2. Never Double Up

Missed sessions are NEVER made up by adding extra sessions in following weeks. Skip and move on. This is a hard rule — never propose "let's do two sessions tomorrow to catch up" or shift missed volume to next week.

### 3. Constraint-Specific Phase Rules

| Constraint | Base | Build | Peak | Taper |
|---|---|---|---|---|
| Equipment | Substitute within ceilings | Same | Same | Same |
| Fatigue | Yellow flag (investigate) | Protect 1 quality session | 1-2 days extra rest is free | Extend taper if not resolving |
| Injury | Cross-train within ceilings | Return to Base 3 intensity if key sport lost >2 weeks | Reassess race goal | Forced rest partially aligns with taper |
| Illness (above neck) | Z1-2 only | Z1-2 only | Z1-2 + strides | Z1 + strides |
| Illness (below neck) | Full stop → Base 1 on return | Full stop → Base 3 for 1 week before Build | Merge into extended taper | Full rest, accept performance hit |

### 4. Duration Thresholds

| Duration | General Action |
|---|---|
| 1-3 days | Skip affected sessions, resume normally |
| 4-7 days | Reframe as recovery week or reduce volume |
| 8-14 days | Roll back 1 week in plan progression |
| 14-21 days | Return to start of current phase |
| 21+ days | Recommend plan regeneration (escalation) |

These are modified by constraint type — illness has stricter ramp-back than equipment for the same duration.

### 5. Ramp-Back Protocols

#### After Injury (affected sport only)
```
Week 1: <20min, Z1, pain-free required
Week 2: +10% volume, Z1
Week 3: +10%, Z1-2
Week 4+: +10%/week, intensity only after pain-free at ≥80% pre-injury volume
```

#### After Below-Neck Illness (all sports)
```
Days 1-2: Walking only
Days 3-4: Easy aerobic, 20-30min, Z1
Days 5-7: Normal aerobic, no intensity
Day 7+: Resume plan from 1-2 weeks prior
Rule: if symptoms return → drop back 1 stage, wait 24h
```

Above-neck illness without fever: no ramp-back needed. Resume normally once symptoms clear, but cap at Z1-2 while symptomatic.

### 6. Cascading Rules

Apply these when `phase_impact` is "structural":

- **Avoid back-to-back easy weeks**: If reducing this week and the NEXT week is already a recovery week, push the recovery week later to avoid two consecutive easy weeks. Swap the recovery with the following loading week.
- **Key sport lost 2+ weeks in Build**: Consider compressing remaining Build and protecting Peak timing. Do not extend Build into Peak.
- **Race date is fixed**: Taper timing is sacred. All cascading must preserve taper start date. Never shift taper or race week.
- **Recovery week overlap**: If the constraint period overlaps with a planned recovery week, merge them — use the constraint rest as the recovery rather than stacking.

### 7. Peak / Taper Illness Rule

When severe illness occurs during Peak or Taper (within ~2 weeks of race):
- Full stop immediately. Arriving healthy matters more than arriving sharp.
- Merge the remaining schedule into an extended taper — do not try to salvage Peak sessions.
- Cap all remaining sessions at Z1/Easy. No intensity before race.
- Include "arriving healthy > arriving sharp" principle in the diff reasoning.

### 8. Equipment Acknowledgment

When the constraint includes available equipment details (e.g., "hotel gym with treadmill and spin bike"), explicitly reference this in the diff. Substitution sessions should use the available equipment — e.g., "ADD: Easy bike on spin bike" or "ADD: Easy run on treadmill."

### 9. Fatigue-Specific Rules

- **Mild** ("just today"): remove today's session if intensity. Keep easy sessions. No structural changes.
- **Moderate** ("all week"): reduce volume ~25%, convert quality sessions to Easy EXCEPT keep exactly 1 quality session (the most important one for the phase — brick in Build, race-pace in Peak). This is the current week only.
- **Severe** ("for weeks"): convert the entire week to recovery. Flag as potential overtraining. If this is a recurring complaint, suggest deeper investigation (sleep, nutrition, life stress).

### 10. Recurring Constraints (Chat History)

If chat history shows a PREVIOUS adjustment for the same constraint type:
- The previous intervention was insufficient. Escalate your response — do not repeat the same reduction.
- Reference the previous adjustment explicitly so the athlete knows you're aware of the pattern.
- For recurring fatigue: consider converting the current week to full recovery, and flag potential overtraining or underlying cause (sleep, nutrition, life stress).
- For recurring injury: reassess whether the sport should remain in the plan at all for this phase.

## Scope Rules

The `phase_impact` field from Step 1 determines how many weeks you modify:

- **"local"**: Modify only the affected week(s). Look at ±1 week for context (avoid back-to-back recovery), but only output diffs for weeks that actually change.
- **"structural"**: Modify affected weeks + all downstream weeks through the end of the current phase. Check if cascading rules apply. Always include a ramp-back schedule when returning from injury or illness.

## Output Format

Output a markdown diff for each affected week. Be precise about what changes and why.

```
## Week N (Phase WX) — MODIFIED
- REMOVE: [day] [type] [sport] [duration] ([reason])
- KEEP: [day] [type] [sport] [duration] (unchanged)
- ADD: [day] [type] [sport] [duration] ([justification with ceiling math])
- CONVERT: [day] [old_type] [sport] → [new_type] [sport] [duration] ([reason])
- Volume: [old_total] → [new_total] ([delta])

## Week M (Phase WY — was Recovery) — RESTRUCTURED
- SWAP: Recovery week pushed to Week M+1 ([reason])
- Sessions: [description of what this week now looks like]
```

### Action Types

- **REMOVE**: Session deleted entirely (injury, illness, missed)
- **KEEP**: Session unchanged — list these so the diff is complete
- **ADD**: New session added as substitute (must show ceiling math)
- **CONVERT**: Session type changed (e.g., Intervals → Easy) but session kept
- **SWAP**: Week function changed (e.g., recovery moved)

### Rules for the Diff

1. Every session in the affected week must appear as REMOVE, KEEP, ADD, or CONVERT. No silent changes.
2. For ADD actions, always show the ceiling calculation: `current [sport] = Xmin, ceiling = X × 1.15 = Ymin, adding Zmin → total Wmin [OK/EXCEEDS]`
3. Volume summary is required for every modified week.
4. If ramp-back applies, include a separate "Ramp-Back Schedule" section after the weekly diffs.
5. If no changes are needed (e.g., constraint doesn't affect this week), do NOT output a diff for that week.

### Ramp-Back Schedule (when applicable)

```
## Ramp-Back: [Sport] Return Protocol
- Week N: [volume], [intensity], [conditions]
- Week N+1: [volume], [intensity], [conditions]
...
- Resume full load: Week N+X (estimated)
```
