# Feature Implementation Plan — DRO-198: Website Landing Page Update

**Overall Progress:** `100%`

## TLDR
Update the already-scaffolded `EmmanuelBreard/dromos-website` repo to reflect the new competitive positioning spec. The original landing page was built against the old spec (pre-audit). This spec covers: dark hero, reordered cards, real screenshots in CSS iPhone frames, new copy, credibility line, race distances, OG meta, sitemap, robots.txt, and amber brand token.

> ⚠️ **This work is in the `EmmanuelBreard/dromos-website` repo — NOT the iOS repo.**
> Clone: `gh repo clone EmmanuelBreard/dromos-website`

## Critical Decisions
- **Tailwind v4 (not v3)** — repo uses `@tailwindcss/vite` plugin + CSS-based config. All custom tokens live in `@theme {}` inside `src/styles/global.css`. There is no `tailwind.config.mjs`.
- **Amber accent via CSS token** — add `--color-amber-400: #FBBF24` and `--color-amber-500: #F59E0B` to `@theme {}` in `global.css`. Reference as `bg-amber-500`, `text-amber-400` in Tailwind classes.
- **ScreenshotMockup: CSS-only iPhone frame** — `rounded-[2.5rem] border-4 border-gray-800 shadow-2xl bg-gray-900` wrapper, no external library, no JS.
- **OG image: sharp script** — add `sharp` as a dev dependency; a one-shot `scripts/generate-og.mjs` creates `public/assets/og-image.png` (1200×630 dark bg + centred app icon). Run once manually; output committed to repo.
- **Sitemap: `@astrojs/sitemap`** — add as a dependency + Astro integration. Requires `site: 'https://getdromos.com'` in `astro.config.mjs`.
- **No changes to `support.astro` or `privacy.astro`** — out of scope for this issue.

    // Run: node scripts/generate-og.mjs
    import sharp from 'sharp';
    import path from 'path';

    await sharp({
      create: { width: 1200, height: 630, channels: 4, background: { r: 17, g: 24, b: 39, alpha: 1 } }
    })
      .composite([{ input: icon, gravity: 'centre' }])
      .png()
